**Kaumei&nbsp;JDBC** gives you the control of hand-written SQL with better safety,
ergonomics, and observability than heavier frameworks.
It keeps SQL front and in focus while compiling the boring JDBC plumbing for you.
This tool is focused on practical, predictable database access for developers
who prefer SQL-centric workflows.

If you cannot wait, start right away with the <Link ref="/intro/installation">installation guide</Link>.
More context before adoption is provided in <Link ref="/intro/introduction">Introduction</Link>.
More details can be found in <Link ref="/intro/why-kaumei">Why **Kaumei&nbsp;JDBC**</Link>


<CardGrid>
	<Card title="Compile-time checks" icon="approve-check">
		Annotation processing generates the JDBC boilerplate code and validates bindings up front.
	</Card>
	<Card title="SQL-first workflow" icon="terminal">
		`@JdbcSelect`, `@JdbcUpdate`, `@JdbcBatch`, and `@JdbcNative` keep queries inline and reviewable.
	</Card>
	<Card title="Integration friendly" icon="extension">
		You can wire **Kaumei&nbsp;JDBC** into Spring, Micronaut, or Quarkus by injecting `JdbcConnectionProvider`.
	</Card>
	<Card title="Modern Java features" icon="sparkles">
		Records, enums, custom converters, and collections like `List` or `Stream` are supported in common cases.
	</Card>
</CardGrid>
